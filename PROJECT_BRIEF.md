# Signal Path — Multi-Path CS Internship Workspace

## One-Sentence Brief

Build an evidence-based, interactive workspace that helps a rising SDSU computer science junior compare and pursue data science, data engineering, software engineering, Java backend, cybersecurity, and ML engineering internships before Summer 2027.

## Honest Career Thesis

The user's Meta connection is useful, but the strongest strategy is not “learn generic data science, then ask for a referral.” It is:

1. target a role family that fits an undergraduate CS profile;
2. build visible proof in SQL, experimentation, Python, and data systems;
3. apply broadly while still learning;
4. use the Meta contact for calibration first and a referral only for a well-matched opening.

The default lane remains product analytics, but every path is a first-class workspace with its own curriculum, projects, resources, interview focus, and application titles.

## Must Have

- Candid role-fit recommendation and referral guidance
- Dated curriculum phases with checkable tasks
- Curated resource library with direct links, price, format, duration, and rationale
- Persistent top-level path tabs for Data Science, Data Engineering, Software Engineering, Java Backend, Cybersecurity, and ML Engineering
- Dashboard, roadmap, projects, resources, and applications as separate workspaces rather than one long page
- Portfolio project briefs with milestones, architecture, proof tests, and resume-ready outcomes
- Internship sprint and lightweight application tracker
- Local progress persistence and export
- Search and filters for courses, videos, books, practice, and communities
- Current SDSU-specific actions and communities
- Responsive, accessible implementation

## Content Rules

- Prefer official course documentation, university material, and primary company sources.
- Treat Reddit and community posts as anecdotal patterns, not hiring policy.
- Do not imply that certificates, referrals, or a single project guarantee employment.
- Do not prescribe every resource; explicitly identify a core route and optional alternatives.
- Show a “last reviewed” date because links and recruiting practices change.

## Visual Direction

An academic mission-control surface for long study sessions: near-black ink, true neutral surfaces, high-visibility vermilion, and a restrained electric mint for completed states. A single route line ties the curriculum together. Dense information should remain calm through strong hierarchy, generous gutters, and compact, familiar controls.

## Success Criteria

The experience is successful when the user can answer all of these in under two minutes:

- Which niche should I test first, and why?
- What exactly should I do this week?
- Which course should I take instead of collecting five alternatives?
- What portfolio evidence do I need before asking for the referral?
- Which internship titles should I search for now?
- How much progress have I made, and what is next?
